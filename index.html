--- layout: default title: Pace Front-End Coding Guidelines ---

Intro

As our team grows and code passes through many hands, we ought to come to an agreement of general guidelines to follow for front-end development.

As with any 'guidelines' these are not meant to be absolute truths, but breaking them must be intentional and with good reason.

Discussion points:

  • Indentation preferences
  • Front-end tooling preferences (frameworks, CLI tools)
  • CSS naming conventions
  • CSS file structure
